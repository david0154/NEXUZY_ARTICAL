## âš ï¸ Error Handling

### Missing Dependencies

If you see this error:

```
Missing Dependency

PDF export requires reportlab library.

Install with:
pip install reportlab
```

**Solution:**
```bash
pip install reportlab
```

---

If you see this error:

```
Missing Dependency

Excel export requires openpyxl library.

Install with:
pip install openpyxl
```

**Solution:**
```bash
pip install openpyxl
```

---

### No Data to Export

If you see:

```
No articles to export
```

**Solution:**
- Create some articles first
- Then try exporting again

---

### File Won't Open

If exported file doesn't open automatically:

**Windows:**
1. Go to `exports/` folder
2. Double-click the file
3. Opens in default PDF viewer or Excel

**macOS:**
1. Navigate to `exports/` folder
2. Double-click the file
3. Opens in Preview (PDF) or Numbers/Excel (XLSX)

**Linux:**
1. Navigate to `exports/` folder
2. Use default file manager to open
3. Or command line:
   ```bash
   xdg-open exports/Articles_*.pdf
   ```

## ğŸ”§ Troubleshooting

### Export button not visible

**Problem:** Can't see export buttons

**Solution:**
- Only visible on **Articles** and **Users** pages
- Must be logged in as **admin**
- Located in top-right of page header

### "Module not found" error

**Problem:**
```
ModuleNotFoundError: No module named 'reportlab'
```

**Solution:**
```bash
pip install reportlab
```

### Excel file won't open

**Problem:** XLSX file created but won't open

**Solution:**
- Install Microsoft Excel, LibreOffice, or Google Sheets
- On Windows: Excel usually pre-installed
- On macOS: Use Numbers or install LibreOffice
- On Linux: Install LibreOffice
  ```bash
  sudo apt install libreoffice
  ```

### PDF looks wrong

**Problem:** PDF formatting issues

**Solution:**
- Update reportlab:
  ```bash
  pip install --upgrade reportlab
  ```
- Try different PDF viewer
